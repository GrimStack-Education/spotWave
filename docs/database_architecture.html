Архитектура базы данных SpotWave

Документ показывает, какие данные хранятся в Prisma/PostgreSQL, какие данные хранятся в Firebase, и как две части системы взаимодействуют между собой.

Обзор Prisma Firebase Взаимодействие Сценарии

Обзор архитектуры

Для SpotWave подходит гибридная архитектура. Firebase отвечает за быстрые real-time данные, а PostgreSQL через Prisma отвечает за надежные структурные данные.

Firebase

  • Авторизация
  • Онлайн-статусы
  • Live-события
  • Участники событий
  • Чаты и сообщения
  • Уведомления

PostgreSQL / Prisma

  • Пользователи и роли
  • Сообщества
  • Категории и интересы
  • Жалобы и модерация
  • B2B-организации
  • Архив событий

Как базы взаимодействуют

Firebase Auth создает пользователя и выдает `uid`. Этот `uid` сохраняется в PostgreSQL в поле `firebaseUid`. После этого backend может связывать Firebase-пользователя с Prisma-моделью `User`.

Что идет в Firebase (Real-time)

  • Сообщения и чаты (chat)
  • Онлайн-статусы (presence)
  • Live-участники событий (event/participant)
  • Быстрые данные профиля (user)
  • Уведомления (notification)

Что идет в PostgreSQL (Structural)

  • Роли и права (UserRole)
  • Структура сообществ (Community)
  • Жалобы и модерация (Report)
  • Справочники (Interest, Category, Venue)
  • Архив событий (EventArchive)

Основные сценарии работы

Регистрация

  1. Пользователь регистрируется через Firebase Auth.
  2. Firebase создает `uid`.
  3. Frontend отправляет Firebase Token на backend.
  4. Backend проверяет токен.
  5. Backend создает запись в PostgreSQL в модели `User`.
  6. Создается документ Firebase в коллекции `user/{userId}`.

Создание события

  1. Событие создается в Firebase в коллекции `event/{eventId}`.
  2. Пользователи сразу видят событие на карте или в списке.
  3. Backend сохраняет копию в PostgreSQL в модель `EventArchive`.
  4. После завершения события PostgreSQL хранит итоговую историю.

Чат

  1. Метаданные чата хранятся в `chat/{chatId}`.
  2. Сообщения хранятся в `chat/{chatId}/message/{messageId}`.
  3. Участники получают сообщения в real-time.
  4. В PostgreSQL попадают только жалобы, модерация и статистика.

Модерация

  1. Пользователь отправляет жалобу.
  2. Backend сохраняет ее в PostgreSQL в модель `Report`.
  3. Модератор принимает решение.
  4. Backend записывает действие в `ModerationAction`.
  5. Если объект находится в Firebase, backend скрывает или удаляет его.

Итог Если данные должны обновляться мгновенно, они идут в Firebase. Если данные должны быть надежными, связанными и административными, они идут в PostgreSQL.

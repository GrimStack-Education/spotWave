## Назначение

Папка `frontend` предназначена для пользовательского интерфейса: страниц, компонентов, состояния экрана, клиентских сценариев и взаимодействия с backend через согласованные контракты

## Правила

- интерфейс не должен содержать серверную логику;
- повторяющиеся UI-элементы лучше выносить в `packages/ui`;
- если логика переиспользуется между проектами, её можно переместить в `packages/shared`;
- документация должна объяснять, как запускать и собирать frontend в конкретном форке;
- визуальные соглашения должны быть едиными внутри приложения

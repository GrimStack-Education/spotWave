## Назначение

Эта папка используется для описания и хранения материалов, связанных с автоматизацией доставки: сборка, проверка, публикация артефактов и деплой. Конкретные инструменты зависят от форка и выбранного стека

## Типовой pipeline

### 1. Проверка изменений

Запускаются базовые проверки качества: форматирование, линтеры, статический анализ и тесты

### 2. Сборка артефактов

Изменения собираются в артефакты, которые можно использовать для тестовых или production-окружений

### 3. Публикация и доставка

Артефакты размещаются в registry, storage или другом целевом хранилище, после чего при необходимости выполняется деплой

### 4. Наблюдение

Pipeline должен оставлять достаточно информации для диагностики: логи, статус шагов и понятные сообщения об ошибках

---

## Правила

- каждый workflow должен иметь понятное назначение;
- сборка не должна смешиваться с ручными действиями без необходимости;
- секреты не хранятся в репозитории;
- условия запуска должны быть явными и воспроизводимыми;
- при изменении процесса нужно обновлять документацию рядом с конфигурацией

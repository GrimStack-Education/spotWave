## Назначение

Эта папка используется для конфигураций веб-сервера: reverse proxy, маршруты, TLS, статические файлы и проксирование сервисов. Конкретная схема зависит от того, какой проект будет построен на основе шаблона

## Типовые сценарии

### Reverse proxy

Nginx может принимать входящий трафик и перенаправлять его в приложение или несколько отдельных сервисов

### Статика

Если часть проекта собирается как статический фронтенд, Nginx может отдавать его напрямую без промежуточного приложения

### TLS

Если проект разворачивается вне локальной среды, конфигурация должна предусматривать безопасную передачу данных и понятный путь подключения сертификатов

---

## Правила

- конфигурации должны быть минимальными и читаемыми;
- одинаковые правила не дублируются в разных файлах без необходимости;
- локальный и production-варианты стоит разделять;
- изменения в маршрутизации должны быть отражены в документации;
- конфигурация не должна содержать секреты

# Infrastructure Docker

Содержит Docker-артефакты для локальной разработки и сборки:

- `docker-compose.yml` — compose-файл для локального запуска (Postgres, backend, frontend).
- `backend.Dockerfile` — сборка и запуск backend-пакета из монорепо.
- `frontend.Dockerfile` — сборка и запуск frontend-пакета из монорепо.

Примечание: в текущей локальной конфигурации сервис `postgres` удалён (локальная БД не поднимается автоматически). Если потребуется поднять Postgres в составе compose — добавьте соответствующий сервис в `docker-compose.yml` или используйте внешний хост/managed БД.

Запуск из корня репозитория:

```bash
# (из корня)
# можно задать имя проекта через env-файл: `infrastructure/docker/.env` содержит `COMPOSE_PROJECT_NAME=spotwave`
docker compose -f infrastructure/docker/docker-compose.yml up --build
```

Замените значения в `DATABASE_URL` и паролях на реальные через переменные окружения или секрет-менеджер.

## Типовая структура

### Образы

Каждый исполняемый модуль может иметь собственный Dockerfile, если ему нужен отдельный runtime или отдельные зависимости

### Compose

Compose-конфигурация должна описывать только воспроизводимый локальный или тестовый запуск. Продакшн-настройки выносятся отдельно

### Переменные окружения

Чувствительные значения не коммитятся. В репозитории хранится только безопасный шаблон `.env.example`

---

## Правила

- образы должны быть минимальными и предсказуемыми;
- кэш сборки используется только там, где это полезно;
- конфигурация должна быть понятна без чтения кода приложения;
- локальный запуск должен быть воспроизводимым с нуля;
- изменения в контейнеризации сопровождаются обновлением README рядом с ними
